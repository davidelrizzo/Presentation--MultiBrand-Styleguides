So what exactly is a front-end Styleguide again?

A reusable library of front-end components (like buttons, tables and grids) that are managed separately to any specific website or app.

LESS overrides allow easy changes

OOCSS & SUITS naming; our way..

  • Preprocessor agnostic (Solves our LESS vs SASS problem)
  • .[prefix]-[Component]-[subElements]--[variant]
    eg. .sg-Radio-indicator--dark
    (Reduces specificity)
  • SS vs CSS (Reduce the cascade!)
  • Styling native tags (eg. <ul>) only in optional file
